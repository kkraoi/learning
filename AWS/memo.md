### 開発インスタンスを通して、本番環境にログイン
```
ssh -i ~/.ssh/practice-aws.pem ec2-user@本番環境パブリックIPアドレス
```
鳥が返ってきたらログイン成功

### puma起動確認
プロセスの確認方法
```
ps aux | grep puma
```

## 開発環境デプロイ
AWSインスタンス画面より、開発環境インスタンスを起動するだけ。

## 開発環境デプロイ停止
1. pumaを停止
2. AWSインスタンス画面より、開発環境インスタンスを停止

### puma停止
開発環境のアプリ階層にて、
```
sudo kill $(cat tmp/pids/puma.pid)
```
