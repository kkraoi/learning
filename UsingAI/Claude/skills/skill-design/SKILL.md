# Skill Design

Claude CodeのSkillを設計・生成するスキル。「○○したい」という入力に対して、本当にSkillが必要か・Promptで十分か・SubAgent向きかを分析し、適切であればSKILL.mdを設計して `~/.claude/skills/<name>/SKILL.md` に保存する。

## Before You Start

- ユーザーの要望が「繰り返し使うワークフロー」か「一度きりの指示」かを見極める
- 既存のSkillが `~/.claude/skills/` に存在しないかGlobで確認する（似た名前のものは更新を検討）
- 要件が曖昧な場合は、具体的なユースケースをユーザーに確認してから判断する

## Subcommands

| Subcommand | Action |
|---|---|
| `analyze` | Skill/Prompt/SubAgentの判断のみ実行（SKILL.md生成はしない） |
| `create <name>` | 指定した名前でSkillを設計・生成する |

## フロー

### ステップ1: 要件の整理

ユーザーのリクエストから以下を把握する。不明な場合は質問する：

- **やりたいこと**: どんな作業を自動化・標準化したいか
- **利用頻度**: 繰り返し使うものか、一度きりか
- **複雑さ**: 単純な指示か、複数ステップのワークフローか
- **スコープ**: 特定プロジェクト専用か、全プロジェクト共通か

### ステップ2: Skill / Prompt / SubAgent の判断

以下の基準で最適な実装方式を判断し、ユーザーに推薦と理由を説明する：

**Promptで十分なケース（CLAUDE.mdや都度指示）:**
- 常時適用したい行動変更（コードスタイル、言語設定など）
- 1〜3行で表現できる単純な指示
- スラッシュコマンドを使わずに自然言語で指示できるもの

**SubAgentが適切なケース:**
- メインコンテキストを汚染したくない独立した調査タスク
- 並列実行したい複数の独立した作業
- 大量の出力が見込まれる探索・分析タスク

**Skillが適切なケース:**
- 明確なスラッシュコマンドとして繰り返し呼び出すワークフロー
- 複数ステップを持つ定型作業
- ドメイン固有の知識・判断基準が必要なタスク
- 複数のサブコマンドを持つ複合ワークフロー

判断が **Prompt** の場合は CLAUDE.md への追記案を提示して終了する。
判断が **SubAgent** の場合は Agent 呼び出しの例を提示して終了する。

### ステップ3: Skill設計（Skillと判断した場合）

以下を決定してユーザーに確認する：

1. **スキル名**: ケバブケース（例: `code-review`, `deploy`）
2. **概要**: 何をするスキルか1〜2文で
3. **Before You Start**: 前提条件・事前チェック（不要なら省略）
4. **サブコマンド**: 2つ以上の明確に異なるモードがある場合のみ定義
5. **メインフロー**: ステップバイステップの実行手順
6. **出力**: 何を生成・変更・報告するか

### ステップ4: SKILL.md生成

確認後、以下の構造でSKILL.mdを生成する：

    # <スキル名>（タイトルケース）

    <スキルの概要。何をするか、どんな時に使うかを1〜2文で。>

    ## Before You Start

    <前提条件のチェックリスト。不要な場合は省略。>

    ## Subcommands（サブコマンドがある場合のみ）

    | Subcommand | Action |
    |---|---|
    | `<subcommand>` | <動作の説明> |

    ## フロー

    ### ステップ1: <ステップ名>

    <具体的な手順>

### ステップ5: ファイル保存

生成したSKILL.mdを以下のパスに保存する：

- **Windows**: `C:\Users\<username>\.claude\skills\<skill-name>\SKILL.md`
- **Mac/Linux**: `~/.claude/skills/<skill-name>/SKILL.md`

保存前にSKILL.mdの内容をユーザーに提示して確認を取る。承認後にディレクトリを作成してファイルを保存し、`/<skill-name>` で呼び出せることを案内する。
