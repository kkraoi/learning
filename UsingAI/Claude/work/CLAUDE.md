# 業務プロジェクト共通ルール
このディレクトリ配下は業務で利用するプロジェクトである。すべての作業において、セキュリティおよび秘密保持を最優先とする。

## 秘密保持
- ソースコード・設計書・仕様書・ログ・設定ファイルなど、業務で知り得た情報を外部へ公開・転載・要約しない。
- 顧客名・会社名・製品名・プロジェクト名・担当者名などの固有名詞は機密情報として扱う。
- APIキー・アクセストークン・パスワード・証明書・秘密鍵・接続文字列などの機密情報を出力・保存・共有しない。
- `.env` や設定ファイルに含まれる秘密情報は表示・引用しない。

## 外部サービスの利用
- 明示的な指示がない限り、業務ファイルの内容を外部サービスへ送信・変換することを前提とした提案は行わない。
- コードやログを外部へ共有する提案を行う場合は、必ず機密情報の除去が必要であることを前提とする。

## ログ・出力
- デバッグ時も個人情報や認証情報を出力しない。
- サンプルコードや説明では、実際の会社情報ではなくダミーデータを使用する。

## ドキュメント作成
- 回答例・サンプル・設計書には架空の会社名・顧客名・URL・メールアドレス・電話番号を使用する。
- 実際の業務情報を例として記載しない。

## Git
- リモートリポジトリの設定変更は提案のみとし、自動では行わない。
- GitHub・GitLab等への公開リポジトリ作成を前提とした提案は行わない。
- 業務コードを公開リポジトリへ移行・コピーする提案は行わない。

## 判断に迷う場合
情報漏えいの可能性が少しでもある場合は作業を停止し、ユーザーへ確認する。機密保持を優先し、利便性のために例外を設けない。
