# スッキリわかるSQL入門 第4版 のメモ

## DBMSについて
- RDBはただのファイル、データベースファイルそのものにSQL文を送るわけではない。
- DBMSという"プログラム"に対して、SQL文を送る。
- DBMSはコンピュータ内で常に稼働していてSQL文を待ち受けている
- ANSI（米国国家規格協会）とISOの標準構文があり、いろんなＤＭＢＳでも共通の構文がある
- "ネットワーク経由"でSQLを送る、それがドライバというソフトウェア
    - javaやcでドライバの命令を呼び出すことでDBMSにSQL文を送信する
    - 通常、SQLクライアントとうSQL文をそのままDBMSに送るソフトウェアが標準で備わっているため、プログラムを作る必要はない

## CHAR と VARCHAR
- `CHAR(10)`: 
    - 7バイトだったとしても、10バイトになる。
    - 格納する量が一定のもの、郵便番号・社員番号
- `VARCHAR(10)`: 
    - 7バイトだったとしても、7バイトのまま
    - 11バイト以上は格納できない
    - データの長さが決まっていないもの

## LIKE 演算子
部分一致検索に使う。

```sql
SELECT * 
    FROM テーブル
    WHERE 列名 LIKE '%検索文字%';
```

パターン文字
- `%`
    - 任意の0文字以上の文字列
    - なんでもいい（文字が無くても、何文字あってもOK！）
- `_`
    - 任意の1文字
    - なんでもいいから、とにかく1文字だけ入る穴埋め
    - 1文字じゃないとだめ、2文字以上はだめ。

## BETWEEN 演算子
範囲の検索に使う

```sql
-- 500 ~ 900 の値のものを探す

SELECT *
    FROM 家計簿
    WHERE 出金額 BETWEEN 500 AND 900;
```

## IN / NOT IN 演算子
- 検索ワードが複数あるときに使う
- 項目リストと一致するかを見る
- `NOT IN` は 検索ワードに含めないときに使う。

```sql
SELECT *
    FROM 家計簿
    WHERE 費目 IN ('食費', '交際費');
```

## ANY / ALL 演算子
- IN とは違い、比較演算子を使う
- ANY は項目リストの検証結果、いずれかTRUEになっているものを返す
- ALL は項目リストの検証結果、すべてTRUEになっているものを返す

```sql
SELECT *
    FROM 家計簿
    WHERE 出金額 > ANY (1000, 2000);

SELECT *
    FROM 家計簿
    WHERE 出金額 > ALL (1000, 2000);
```

## SELECT の修飾

### DISTINCT
- 検索結果の重複を削除して返す
- "明確な"
```SQL
SELECT DISTINCT 列名...
    FROM テーブル名
```

### ORDER BY
- 並び替え、昇順・降順
- デフォルトは昇順（ASC）
- 対象の列が複数あってもOK、優先順をつけられる
- ORDER BY 句をつけないと実質的にランダムの並びになる

```sql
SELECT * 
    FROM 家計簿
    ORDER BY 出金額 DESC, 入金額 DESC
```

### OFFSET - FETCH
- トップ5を取り出したい、3番めを取り出したい、という要望に答える
- 言語によって構文が乱立している
- 大抵ORDER BY句と一緒に用いられる

トップ5を取り出したい
```sql
SELECT 費目, 出金額
    FROM 家計簿
    ORDER BY 出金額 DESC
    OFFSET 0 ROWS -- 上から0番目を起点にする
    FETCH NEXT 5 ROWS ONLY; -- そこから5番目まで取り出す
```

3番めを取り出したい
```sql
SELECT 費目, 出金額
    FROM 家計簿
    ORDER BY 出金額 DESC
    OFFSET 2 -- 2番めを起点にする
    FETCH NEXT 1 ROWS -- そこから１番目をとりだす
```

### UNION
- 和集合
- 列の種類・数が同じであれば、検索結果をガッチャンコできる
    - 同じじゃない場合は、列数の少ない方の値にNULLで揃える
- ORDER BY句は、最後に書く
    - 列の指定は、１つ目のSELECTの列に準拠する

家計簿テーブルと家計簿アーカイブテーブルを合わせる
```sql
SELECT 費目, 入金額, 出金額 FROM 家計簿
    UNION
SELECT 費目, 入金額, 出金額 FROM 家計簿
    ORDER BY 2, 3, 1;
```

列の種類・数が同じではないパターン
```sql
SELECT 分類, 商品名, サイズ, 単価 FROM 注文履歴
    WHERE 分類 = '1'
UNION
SELECT 分類, 商品名, NULL, 単価 FROM 注文履歴
    WHERE 分類 = '2'
UNION
SELECT 分類, 商品名, NULL, 単価 FROM 注文履歴
    WHERE 分類 = '3'
ORDER BY 1, 2;
```

### EXCEPT
- 差集合
- 検索結果から検索結果を引いて返す
- 使用例: 今月の費目 - 先月の費目 から、今月初めて登録する費目を抽出できる
- UNION と違って順番が重要

今月初めて登録する費目を抽出
```sql
SELECT 費目 FROM 家計簿
EXCEPT
SELECT 費目 FROM 家計簿アーカイブ
```

### INTERSECT
- 積集合
- 検索結果の重複を返す
- UNIONと同様に順は関係ない

先月と今月の重複を抽出
```sql
SELECT 費目 FROM 家計簿
INTERSECT
SELECT 費目 FROM 家計簿
```

## CASE演算子
- 選択列が、aのときはA, bのときはB、それ以外はZ って表現できる

```sql
SELECT 費目, 出金額,
    CASE 費目 
        WHEN '居住費' THEN '固定費'
        WHEN '水道光熱費' THEN '固定費'
        ELSE '変動費'
    END AS 出費の分類
    FROM 家計簿 
    WHERE 出金額 > 0;


SELECT 費目, 入金額,
    CASE 
        WHEN 入金額 < 5000 THEN 'お小遣い'
        WHEN 入金額 < 10000 THEN '一時入金'
        WHEN 入金額 < 30000 THEN '給料出た'
        ELSE '想定外の収入です'
    END AS 収入の分類
    FROM 家計簿
    WHERE 入金額 > 0;
    
```

## 文字列の一部を抽出する関数
```sql

/* 費目列から1~3文字で「費」がつく行を取り出し。 */
SELECT * FROM 家計簿
    WHERE SUBSTRING(費目, 1, 3) LIKE '%費%';
```

## 文字列を連結する関数
- 通常は「+」や「||」を使って文字連結する
- 関数と演算子はスピードは変わらないが、NULLの取り扱いに違いがある
    - 関数はNULLでもかえしてくれる
    - 演算子はNULLだと返さない

```sql
SELECT CONCAT(費目, ':' || メモ) FROM 家計簿

-- これと同じ結果
SELECT 費目 || ':' || メモ FROM 家計簿
```

## 指定桁で四捨五入する関数
```sql
-- 10の位を四捨五入して丸める
SELECT 出金額, ROUND(出金額, -2) AS 百円単位の出金額
    FROM 家計簿
```

## 指定桁で切り捨てる関数
```sql
-- 10の位を切り捨てして丸める
SELECT 出金額, TRUNC(出金額, -2) AS 百円単位の出金額
    FROM 家計簿
```
