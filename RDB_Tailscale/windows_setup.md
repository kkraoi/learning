# Windows側セットアップ手順

対象: Windows 11 Pro 25H2

## STEP 1. Tailscaleアカウントを作る

1. ブラウザで https://tailscale.com/ を開く
2. 右上「Get started」→「Sign up」
3. Google / Microsoft / GitHub いずれかのアカウントでサインアップ（新規パスワード管理を増やしたくなければGoogleアカウント推奨）
4. サインアップ後、管理コンソール（Admin Console, `login.tailscale.com/admin/machines`）が開ければ完了。まだ何も端末が登録されていない状態でOK。

> 無料の Personal プランは 3ユーザー・100台まで利用可能。個人の2台構成では制限に一切引っかからない。

## STEP 2. WindowsにTailscaleをインストール

1. https://tailscale.com/download/windows を開く
2. 「Download Tailscale for Windows」をクリックしてインストーラ（`tailscale-setup-x.xx.x.exe`）を実行
3. インストーラの指示に従いインストール（管理者権限を求められたら許可）
4. インストール完了後、タスクトレイ（画面右下）にTailscaleアイコンが出現する
5. アイコンをクリック →「Log in...」→ ブラウザが開くのでSTEP1で作成したアカウントでログイン
6. ブラウザ側で「Connect」を承認するとタスクトレイのアイコンが緑色（接続済み）に変わる

## STEP 3. Windows機のTailscale情報を確認する

1. タスクトレイのTailscaleアイコンをクリックすると、自機のTailscale IP（`100.x.x.x`形式）とマシン名が表示される
2. あるいはPowerShellで確認:
   ```powershell
   tailscale status
   tailscale ip -4
   ```
3. 表示された **マシン名**（例: `desktop-abc123`）と **Tailscale IP**（例: `100.101.102.103`）を控えておく。Mac側の接続設定で使う。

### MagicDNSを有効化しておく（推奨）

IPアドレスを覚えなくても `マシン名.tailnet名.ts.net` のホスト名で接続できるようになる。

1. 管理コンソール（`login.tailscale.com/admin/dns`）を開く
2. 「MagicDNS」を **Enable** にする

## STEP 4. リモートデスクトップ(RDP)を有効化する

1. `Windows設定` を開く（`Win + I`）
2. 「システム」→「リモートデスクトップ」
3. 「リモートデスクトップ」のトグルを **オン** にする
4. 確認ダイアログが出たら「確認」をクリック
5. 「リモートデスクトップ ユーザー」をクリックし、接続を許可するユーザーを確認（通常は現在ログイン中のユーザーが自動で許可されている）
   - 別ユーザーを追加したい場合はここで「追加」から選択
6. 「詳細設定」の項目にある **「ネットワークレベル認証(NLA)を要求する」は有効のままにする**（セキュリティ上重要。無効化しない）

### Windowsアカウントにパスワードを設定する（未設定の場合は必須）

RDPはパスワード未設定のアカウントには接続できない。

1. `設定` →「アカウント」→「サインイン オプション」
2. 「パスワード」→ 未設定なら「追加」して強固なパスワードを設定

## STEP 5. スリープ・自動ロックの設定を変更する

外出中にWindowsがスリープすると接続できなくなるため、電源設定を変更する。

1. `設定` →「システム」→「電源」
2. 「画面とスリープ」で以下をすべて **「なし」** に変更:
   - 電源接続時に、次の時間が経過後デバイスをスリープ状態にする
3. ノートPCではなくデスクトップ運用前提（常時電源接続）であることを確認

> 画面がオフになるのは問題ない（RDPは画面オフでも動作する）。問題になるのは「スリープ/休止状態」に入ること。

## STEP 6. ファイアウォールの確認

Windows標準のファイアウォールは、リモートデスクトップ有効化時に自動的に必要なルール（ポート3389, TCP/UDP）を作成する。通常は追加作業不要。

念のため確認する場合:

1. `コントロールパネル` →「Windows Defender ファイアウォール」→「詳細設定」
2. 「受信の規則」で `リモート デスクトップ` を検索し、有効になっていることを確認

> **重要**: ここでポート3389をルーターで外部公開（ポートフォワーディング）する必要は **ない**。Tailscaleのプライベートネットワーク経由でのみアクセスするため、ルーター側の設定は一切不要かつ行わないこと（外部に晒すと総当たり攻撃の標的になる）。

## STEP 7. Windows側の作業完了チェックリスト

- [ ] Tailscaleにログイン済み、タスクトレイのアイコンが緑
- [ ] Tailscale IP または MagicDNS名を控えた
- [ ] リモートデスクトップが有効
- [ ] ログインユーザーにパスワードが設定されている
- [ ] スリープが無効化されている（電源接続時）
- [ ] ネットワークレベル認証(NLA)が有効なまま

ここまで完了したら [mac_setup.md](./mac_setup.md) に進む。
